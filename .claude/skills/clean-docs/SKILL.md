# Documentation Cleanup Skill

This skill helps clean up verbose, repetitive, or poorly organized documentation files by:

1. **Removing redundancy**: Eliminate repetitive sections and consolidate duplicate information
2. **Improving clarity**: Simplify complex or verbose explanations
3. **Better organization**: Restructure content for logical flow
4. **Consistency**: Ensure consistent tone, formatting, and terminology
5. **Conciseness**: Remove unnecessary words while preserving meaning

## Usage

Invoke this skill on documentation files (markdown, text, etc.) that need cleanup:

```
/clean-docs path/to/doc.md
```

Or specify multiple files or patterns:

```
/clean-docs docs/*.md
```

## Process

1. **Read** the documentation file(s)
2. **Analyze** for:
   - Repetitive content
   - Verbose explanations
   - Poor structure
   - Inconsistent formatting
3. **Clean** by:
   - Consolidating duplicate information
   - Simplifying verbose sections
   - Reorganizing for better flow
   - Standardizing formatting
4. **Preserve**:
   - Technical accuracy
   - Important details
   - Code examples
   - Links and references

## What to Clean

- Remove filler words and redundant phrases
- Consolidate repeated concepts into single, clear explanations
- Break long paragraphs into digestible chunks
- Use bullet points for lists instead of prose
- Remove outdated or irrelevant sections
- Simplify complex sentence structures

## What to Preserve

- Technical accuracy and precision
- Code examples and outputs
- Links, references, and citations
- Unique information in each section
- Important warnings or notes
